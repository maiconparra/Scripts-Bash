# Guia de Contribuição

1. Sempre escreva scripts Bash compatíveis com ShellCheck.
2. Use funções utilitárias de utils/ sempre que possível.
3. Documente toda nova funcionalidade em docs/.
4. Teste localmente antes de abrir PR.
5. Não suba arquivos .deb, logs ou temporários.
6. Siga a arquitetura modular descrita em docs/ARCHITECTURE.md.
