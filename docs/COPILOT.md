# Guia para GitHub Copilot e Assistentes

## Como acionar
- Sempre consulte docs/ antes de sugerir mudanças.
- Siga as convenções de docs/ARCHITECTURE.md e docs/CONTRIBUTING.md.
- Responda em português (pt-br).
- Nunca sugira versionar arquivos .deb, logs ou temporários.
- Use ShellCheck e boas práticas Bash.

## Fluxo de trabalho
1. Leia docs/ para contexto e padrões.
2. Siga a arquitetura modular.
3. Documente toda alteração relevante em docs/.
4. Priorize clareza, modularidade e compatibilidade Ubuntu.
